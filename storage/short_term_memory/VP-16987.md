## ROOT CAUSE (確定)
`base-report.service.ts` generateAndUploadReports 用 naive `line.split(',')` 切 csvReport 回傳的 CSV，`report_release_time: new Date(columns[12])` 取錯欄位（同一 accession 2601026475 重複多列、值像 170/1450/260/900 數字片段 → 欄位對位錯，疑 CSV 內含逗號/引號使 split 位移，或欄位索引假設過時）。產生 year<1000 的 Date → MySQL DATETIME 拒絕 → `createMany` throw → per-customer catch 靜默吞 → **periodic_report_records 永遠空**。

**上傳 vs 寫紀錄順序**：upload 在 createMany 之前 → 檔案會送出，但 record 寫不進。故「client 收不到」的歷史症狀，不能單由此 bug 完全解釋（此 bug 下檔案仍會送達）；3/31 當下可能 (a) 部署的 code 不同/upload 也失敗、(b) csvReport 回空、(c) pod 當下沒跑。3/31 log 已 GC 無法回溯。但此 bug 是真實可重現、且解釋了空表。
