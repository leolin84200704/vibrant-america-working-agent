## Lessons Learned

### [2026-05-11 19:35]
- **Helper pattern for cross-cutting RPC concerns**: 大 migration（OAuth2、auth header、tracing）寫一層 helper 包 secret/token logic，caller 只描述「我需要哪些 extra field」，未來 cross-cutting policy 改變只需動一處。
- **Optional context migration (Strategy A)**: 改 service signature 加 optional context、caller 完全不動 → 每個 commit 行為等同、回滾粒度小。Phase 1a 全 service 加 hook，Phase 1b 才實際 thread caller。
- **Resilient helper > Strict helper**: helper 內部 try/catch 並 return undefined，比起 throw 更適合 migration 期間（caller 已有 fallback path）。Logger.warn 留痕。
- **Pre-existing spec bug 三種來源**：
  1. 加 DI 後 TestingModule 缺 mock provider — 補 `provide: X, useValue: stub`
  2. spec 用 `new ClassName(args)` 直接 instantiate 時，原本就漏 constructor arg（DI 不會撞到 path）— 加新 inject 順手補
  3. spec expectation 跟 service 行為脫節（service 有意改了 spec 沒同步）— `git log -S '<expected_token>'` 找改動 commit，看 commit intent 決定改 service 還是改 spec
- **找 schema 真相用 multi-site cross-check**: 同 field 在 repo 多處被用，分歧（7:1）+ dto 註解 = 真相方向；再加 production probe 驗證更穩。
