## Ticket Analysis

### [2026-06-22]
**Type**: 新功能 Story (P2 Medium, Dev To Do, assignee=Leo, reporter=Xiaoye Li, PRD author=Junjie Lin)
**Summary**: [BE] Daily Appointments Report — 排程 job 產生每日 appointments XLSX 並 email 給 clinical team。
**目的**: VI My Calendar 停機時的備援。歷史上 IT 每週一~五 12:01 AM 從 CRM calendar 寄出，現在 scheduling 搬到 VI/UniMod，要 restore 這個 safety net。

**AC / FR 重點**:
- Mon–Fri 12:01 AM PT 排程
- 讀 **upstream scheduling 資料源**（獨立於 VI My Calendar / UniMod 前端 — 前端掛掉也能產生）
- XLSX 每個 clinical team member 一個 tab（目前 9 位），依 start time 升冪
- 所有 event category（Zoom Meeting/Call/Sales Event/Task/Client Inquiry/block/coverage）
- 各 member local timezone（PDT/EDT/MDT）
- 19 個 legacy 欄位（含 Details bracketed sub-fields）
- email .xlsx 附件，全自動
- job 失敗 → alert IT/Engineering
- FR-8: content scope 須符合 Privacy review 結果

### 程式碼調查結論 (explore agents)
**LIS-transformer-v2**（推薦 home）:
- 擁有 calendar 資料源：`v2_event`/`v2_calendar`/`v2_event_participant` (calendar_prod schema)
- 已有 cron 基礎設施：`@nestjs/schedule` 已註冊；`inbound-sync.scheduler.ts` 用 `@Cron('0 1 * * *', {timeZone:'America/Los_Angeles'})` — 正是需要的 PT-cron pattern
- 已有 email + 附件支援：`email.service.ts` 透過 Azure Kafka topic `notification-email-template` 送 Postmark，`AttachmentType{Name,Content(base64),ContentType}` 支援附件
- `reminder.service.ts` 已示範查 practice 150105 events by participant + per-member timezone resolution (`recipient.timezone||event.timezone||'America/Los_Angeles'`)，`CLINICIAN_PRACTICE_ID=150105`
- **缺**: 無 xlsx library（需加 exceljs/xlsx）；calendar DB 無 phone/CRM/provider name 欄位

**lis-backend-emr-v2**（僅作 XLSX builder 參考，非 home）:
- 成熟的 `scheduled-reports/base-report.service.ts`: XLSX 多 sheet builder (`XLSX.utils.book_new/aoa_to_sheet/book_append_sheet`)、`@Cron` + POD_ROLE/ENVIRONMENT guard、report record table
- 但無 calendar 資料、無 email 交付（只有 SFTP）、領域不對

## Approaches Considered

### [2026-06-22]
**方案 A（推薦）**: 在 LIS-transformer-v2 新增 daily-report 排程
- 新 service in `src/calendar/`，`@Cron('1 0 * * 1-5',{timeZone:'America/Los_Angeles'})` (Mon-Fri 00:01 PT)
- 查 v2_event (practice 150105, 當日, by participant) → 依 member 分組 → 各自 timezone 渲染 → exceljs 產 XLSX 多 tab → base64 → email.service 送附件
- 缺漏的 CRM/provider/phone 欄位走既有 Customer gRPC client 補（或留空，視 legacy sample）
- Pros: 資料源 + email infra + cron 都在同 repo，最少跨服務；reminder.service 已有 80% 查詢範本
- Cons: 需加 xlsx 依賴；CRM 欄位來源未定；calendar timezone 資料品質風險（見 VP-16924/16945 'UTC' sentinel 問題）

**方案 B（不推薦）**: 在 emr-v2 做，跨 gRPC 取 calendar 資料 — 領域錯置、email 未實裝、需新 gRPC 介面，成本高。

## Decisions Made

### [2026-06-22] Leo 確認的 scope
- **Repo**: LIS-transformer-v2 ✓
- **Clinical team member / tab / 收件人**: `v2_calendar` WHERE `practice_id=150105` AND `role != 'patient'`（注意：schema 欄位是 `practice_id` 不是 `clinic_id`，role enum=role_type: clinicadmin/provider/patient）。每位一個 tab，email 寄給其 `calendar_owner_email`
- **Customer（每筆 event 的客戶）** = event 的 participant 中 role=patient 那位 → 取其 `calendar_owner_firstname + lastname`
- **欄位 mapping（19 欄保留 header，只填以下，其餘留空）**:
  - Customer_provider_name = patient participant 的 calendar_owner_firstname + lastname
  - Category = `v2_event.event_type`（直接用，不做分類對應）
  - Details = `v2_event.notes`
  - From = `v2_event.start_time`（member local tz）
  - To = `v2_event.end_time`（member local tz）
  - Created_at = `v2_event.create_time`
  - （Title=event_title, Clinical_team_member=member name 待確認是否填）
- **Privacy review (FR-8)**: Leo 自己跟 PM 確認，暫不處理
- **告警 (FR-7)**: 每日信 cc `hung.l@zymebalanz.com`；job 失敗時寄信給 `hung.l@zymebalanz.com`（Leo 靠有無收到信判斷）
- **legacy sample**: 有了 mapping 後不再 blocking

### Email 基礎設施事實（email.service.ts 調查）
- `sendEmail(type, params)` 是 **template-based**：每個 notification_type 綁死 Postmark TemplateId（patient 38719290 / non-patient 38777577），且 `Cc` 在 buildEmailPayload 硬寫死 `''`、`To` 單一字串
- `sendBuiltMessage(payload)` 可送**任意自組 payload** 到 Kafka topic `notification-email-template`，payload 形狀含 To/Cc/Bcc/Subject/TemplateId/TextBody/Attachments/TemplateModel
- → 送「報告附件 + cc hung.l」要自組 payload 走 sendBuiltMessage；但仍需一個 Postmark TemplateId 或確認 downstream consumer (LIS-setting-consumer) 接受 template-less 的 TextBody/HtmlBody 純信。**此為唯一待驗證技術點**（實作時自查）
- 附件格式：`AttachmentType{Name, Content(base64), ContentType}`，XLSX ContentType = `application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`
