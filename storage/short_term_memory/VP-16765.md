## Lessons Learned

### [2026-05-27]
1. **判斷 hl7_file_input 失敗單由 emr-v2 還是 Java cron 處理**：emr-v2 processor 遇 customer_not_found 會設 `parse_finished=true`（不再重試）；若 `parse_finished=0` + customer_not_found 且 retry_num=0 → 是 **Java cron** 處理（該 client 未在 emr-v2 cutover batch）。reprocess 用 `SET retry_num=3` 對 Java cron 有效。
2. **emr-v2 `retry_num` 是從 `INITIAL_RETRY_NUM=5` 倒數的剩餘次數**（`hl7-order-fetch.service.ts`），=0 代表耗盡；emr-v2 fetch 是 SFTP 掃檔驅動、不重掃 hl7_file_input。
