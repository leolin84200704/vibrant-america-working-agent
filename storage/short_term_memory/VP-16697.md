## Lessons Learned

1. **Ticket 圖片用 atlassian media blob URL 過期問題**：description 內 `blob:https://media.staging.atl-paas.net/?type=file&localId=...` 拿到的瞬間 URL，過幾天 / 重 query 就 expire 取不到圖。**遇到 ticket 帶圖但 blob URL 拿不到時，第一動作是請 Leo 重新貼圖或從 Jira UI 取 attachment 直連**，不要憑文字推測 UI。
2. **Zoom OAuth 流程不主動 sync events**：`zoom.service.ts:exchangeCodeForTokens` 只做 token storage + 寫回 zoom_calendar_ids（fail-safe）。Backend 沒有 listMeetings endpoint — frontend 不可能透過 backend 拉 Zoom personal meetings。
3. **v2_third_party_integration vs v2_calendar.zoom_calendar_ids 可能 desync**：`updateCalendarZoomId` 在 try/catch 內，若 `getZoomUserId` (Zoom /users/me API) 失敗，token 已存但 zoom_calendar_ids 維持空。Suzette 就是這種狀態。下次驗證 connection 健康度要兩個 table 一起看。
