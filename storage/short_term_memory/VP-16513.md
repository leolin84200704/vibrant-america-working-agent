## Ticket Analysis
- [BE] Meeting Type CRUD - provider-level. Epic VP-15817. LIS-transformer-v2 calendar. 站在 VP-16512 的 v2_meeting_type 上。
- 目的：讓 provider/practice admin 後台增刪改查自己的 meeting type。
- 改前：v2_meeting_type 空表、無任何 API。改後：一組 REST CRUD（auto slug + booking url / list active-default / partial update / soft delete / ownership）。

## Decisions Made
- ownership：owning provider(customer_id==providerId) 或 practice admin(同 clinic，isAdmin 或 v2_calendar.role='clinicadmin')。provider_id=customer_id、practice_id=clinic_id。
- booking_url：沿用 PublicBookingService.generateBookingUrl + /{slug}（mypatienthubs.com）。⚠️ FE VP-16519 寫 healmeet.com，網域待 FE/PM 確認。
- slug：自實作 sanitize + dedup（-2/-3），rename **不**重新產 slug（保連結穩定）。
- availability 寫入不在此票（VP-16516）；GET :id 讀回 weekly/date 子表（目前空）。

### 風險與邊界案例
- 並發 create 同名 meeting type：generateUniqueSlug(dedup) 與 prisma.create 非 atomic，兩個 request 可同時算出同一 slug，其一撞 @@unique([provider_id,slug]) 拋 Prisma P2002 → 目前 service 未 catch → 回 unhandled 500 而非 409/friendly error。建議 create 包 try/catch P2002 retry 或回 ConflictException
- generateUniqueSlug (L94-95) 用 startsWith(base) 撈既有但未過濾 is_deleted=false：soft-deleted row 仍佔用其 slug，重建同名 meeting type 會永遠拿到 -2 suffix（slug accretion）。與 list/getOne 恆排除 is_deleted 的行為不一致 — 需 PM 確認 slug 是否該在 soft-delete 後釋放
- name 只有 @IsString+@MaxLength，未擋空字串/純空白；slugify 後若為空會 fallback 'meeting-type'，但 name 本身可存成空白 — 建議加 @IsNotEmpty 或 trim 後長度檢查
- update (PUT) 允許改 is_active，與 VP-16514 的 PATCH :id/status 功能重疊；無害但 FE 契約需明確走哪條
- booking_url 網域 mypatienthubs.com vs FE 票可能預期的 healmeet.com 不一致：FE 接上後 URL 會對不齊（已驗證 source 確為 mypatienthubs.com），需 PM/FE 拍板
- reorder/toggleStatus(VP-16514) 一併在同 PR：若 Leo 想 VP-16513 與 VP-16514 分開 review/merge，需拆 PR 或在 Jira 註明同 PR 交付兩票
- authorization 邊界：assertProviderAccess 對 patient-payload JWT(無 customer_id/clinic_id) 應落入 403 而非洩漏 — QA 需驗 patient token 打 provider endpoint 回 403
