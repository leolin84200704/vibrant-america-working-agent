## Ticket Analysis

### [2026-06-18]
- **Title**: EHR Integration Request — validation error message returned in raw format instead of user-friendly string
- **Type**: Bug, P2 Medium, assignee Leo, status Dev To Do, reporter Xiaoye Li
- **症狀**: 送 EHR Integration Request，NPI 非 10 碼時，前端顯示 `{"clinicInfo.Customer NPI must be exactly 10 digits"}`，期望 `Customer NPI must be exactly 10 digits.`
- **端點**: POST `/api/v1/integration-management/auto-integrate/requests`，DTO `CreateIntegrationRequestDto`（nested `ClinicInfoDto`）。
- **驗證**: `create-integration-request.dto.ts:233` `@Matches(/^\d{10}$/, { message: "Customer NPI must be exactly 10 digits" })`，customerNpi 在 nested `ClinicInfoDto`（`@ValidateNested()`+`@Type(()=>ClinicInfoDto)`）。

### Root cause（真實重現，非推測）
用真實 ValidationPipe（main.ts 設定 whitelist/forbidNonWhitelisted/transform，**無 exceptionFactory**）對 bad NPI 跑 transform，後端實際回：
```
{"message":["clinicId must be a numeric string","clinicId must be a string","clinicInfo.Customer NPI must be exactly 10 digits"],"error":"Bad Request","statusCode":400}
```
→ **NestJS 11 預設 ValidationPipe 對 nested DTO 會加 `clinicInfo.` property-path 前綴**（這是「不 user-friendly」的後端來源）。`{...}` 大括號則是**前端**把 message 陣列原樣 render。
- 全 repo 無 APP_FILTER/APP_INTERCEPTOR/@Catch/自訂 exceptionFactory；controller 無手動格式化錯誤。

## Approaches Considered

### [2026-06-18]
後端修法（去掉前綴、給乾淨訊息）：
- **A. 全域 exceptionFactory（main.ts）**: 遞迴攤平 nested error、回乾淨 constraint 訊息（去 `prefix.`）。一致但**blast radius 大**（影響 emr-v2 所有 validated endpoint 的錯誤格式，其他前端可能依賴現格式）。
- **B. controller-scoped ValidationPipe**: 只在 integration-request controller(s) 掛自訂 exceptionFactory。blast radius 最小、targeted。
- 前端面：即使後端訊息變乾淨，前端仍把 array 顯示成 `{...}` → 需前端把 message join 成字串。屬 ehr-frontend，非 emr-v2，需確認 owner。

待 Leo 定 scope（A/B + 是否含前端）。

## Decisions Made

### [2026-06-18] Leo 定案: **B (scoped 在 integration-request controller)** + **只修後端，前端另論**。
關鍵實作洞察: 全域 ValidationPipe 先跑先 throw → 再掛 scoped ValidationPipe+exceptionFactory **無效**。正解 = controller-scoped **exception filter** 攔截全域 pipe 丟的 400 並清前綴。

### [2026-06-18] branch 待建; 變更:
- 新增 `auto-integrate/filters/validation-error-cleanup.filter.ts`: `@Catch(BadRequestException)` filter，把 `message[]` 內 `^(?:ident\.)+` nested 前綴 strip 掉（"clinicInfo.Customer NPI..." → "Customer NPI..."）。非陣列 message / 一般 BadRequest 原樣放行；保留 response shape (statusCode/message[]/error)。
