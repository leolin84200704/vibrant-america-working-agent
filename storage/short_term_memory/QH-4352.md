## Ticket Analysis

**Goal（目的）**：QH-4352 是 QA Task，由 BE Story VP-16014（issuelink type="Test"/"is tested by"，VP-16014 已 Done）自動產生。目的是「驗證已上線的行為」：Settings > EMR Integration 的 vendor dropdown 只回傳 public 可用 vendor，排除 private/test/inactive，且 vendor 公開與否由 DB ehr_vendors.is_public 動態維護（新增 vendor 不需改 FE，source of truth = Notion EMR Vendor List）。沒有新 BE feature code 要寫；ticket 停在 To Do 是 QA 尚未跑驗證，非技術阻塞。

**Affected repos**：lis-backend-emr-v2

### 建議方案（改前 / 改後為何有效 / 改什麼）
不建 branch、不改 feature code（已驗證 PR #76 merged，commits eb50377/373bc97，migration 目錄 prisma/migrations/20260421_add_is_public_to_ehr_vendors/ 存在，FE VP-16261 Done）。先向 Leo 確認「是你親自跑驗證回填，還是掛名由 QA 執行」。

改前現狀：findAll() 只過濾 is_active=true，無 public/private 機制，VEJO/ZymeBalanz/MDHQTEST 等 active 即出現在 dropdown。

改後為何有效（已實作，逐行核對無誤）：
- prisma/schema.prisma EhrVendor model 新增 is_public Boolean @default(true) + index；data migration 採「全部設 FALSE → 明確設 public TRUE」安全模式（prod 33 vendors/18 public/15 private；staging 11/6/5）。
- src/modules/integration-management/vendor-management/dto/vendor-query.dto.ts:45-56 新增 optional isPublic query param，'true'/'false' string transform。
- src/modules/integration-management/vendor-management/services/ehr-vendor.service.ts:106-110 filters.isPublic !== undefined 時用其值，否則 default where.is_public=true → 無參數=只回 public，向後相容。
- controller ehr-vendor.controller.ts:96 findAll(query, user.isAdmin)；isAdmin 只影響 SELECT 欄位（SFTP secrets），不影響 is_public/is_active 列過濾。

驗證做法（live-not-mock，依 iron rule 用真實 staging dist endpoint + DB）：對 GET /api/v1/integration-management/vendor-management/ehr-vendors 驗 5 case — (1) 無參數→只回 public（count 對齊 DB SELECT COUNT(*) WHERE is_public=1 AND is_active=1）；(2) ?isPublic=false→只回 private；(3) ?isPublic=true 顯式=與 (1) 相同；(4) VEJO/ZymeBalanz/MDHQTEST 不出現在無參數結果；(5) 回傳 name capitalization 對齊 Notion。比對方式：SELECT code,name,is_public,is_active FROM ehr_vendors 全表 dump，與 API 回傳逐 row 比對（不抽樣）。

唯一潛在真實工作量：若 Notion vendor list 自 4/21 migration 後新增/變動 vendor，需 bounded SQL 補設 is_public（明列 vendor code、time/IDs 界定、UPDATE 後 100% 反查驗證、prod 寫入遵 iron rule）。

### 風險與邊界案例
- Admin 隱藏 consumer：controller findAll 傳 user.isAdmin 只改 SELECT 欄位，不改 is_public 列過濾 → 即使 admin 不帶 isPublic 也只看到 public vendor。若 admin panel 依此 endpoint 管理 private vendor，現在需顯式帶 ?isPublic=false，否則 silently 看不到 private。需確認 admin panel 是否為 consumer。
- is_public 欄位 @default(true)，但 migration 是「全 FALSE 再設 public」；兩者方向相反 → 之後新建的 vendor（走 create()，未在 migration 範圍）會自動 is_public=true 進入 dropdown。若新 vendor 實為 private 會 silently 出現，需 QA 留意。
- ehr_vendors.code 是 mixed case（ElationEMR/ELATIONEMR、ChARM_EHR、OptimalDX），寫驗證 SQL 用實際 DB 值，勿假設全大寫；MySQL case-insensitive collation 比對可能掩蓋差異。
- 已選 private vendor 的既有 practice 不受影響（只是 dropdown 看不到該選項）— 應驗證既有 integration 仍正常運作，未被過濾掉。
- Notion（source of truth）與 DB 手動同步會 drift；自 4/21 起若有新 vendor 未補 is_public 則 dropdown 缺項且不報錯。
- search param 與 is_public default 疊加：帶 search 但不帶 isPublic 時仍只搜 public，QA 需確認此為預期行為。
