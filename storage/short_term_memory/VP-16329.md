## Lessons Learned
### [2026-04-27 23:40]
1. **Same-practice add-provider 必須先 query `customer_id IN (...)` 對比 ticket 列表**：
   - ticket 列出的 provider 可能已存在（如 VP-16329 的 8445）
   - 重疊者走 UPDATE，全新者走 INSERT
   - LTM 應加：「Step 5a probe 必須包含 ticket provider 與 DB existing 的交集檢查」
2. **insert-ehr-integration.ts unique constraint = (customer_id, clinic_id, ehr_vendor_id)**
3. **VP-16280 + VP-16329 連續驗證 same-practice 全 align pattern**：既有 RESULT_ONLY → FULL_INTEGRATION + MSH=Practice ID + archive_path 統一。下次同類 ticket Plan 階段就應預設這套處理，不要等 Leo confirm 才提
4. **跑 npx ts-node script 看結果用 grep 過濾**「Successfully|Error|❌|✅」，不要用 tail 截取
