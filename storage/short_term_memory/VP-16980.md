## Ticket Analysis
### [2026-06-15 21:29]
[BE] Grant Sales Director Role access to EMR Integration Request APIs. P1-High. Reporter Xiaoye Li, assignee Leo.
- 現況: SD token 打 /lis/emr-service/api/v1/integration-management/auto-integrate/requests → 403 "Access denied: no customer or clinic permissions"。
- AC1 SD 可呼叫 requests 及相關 endpoint; AC2 retrieve/approve/reject; AC3 不擴及 EMR integration management 以外的 API。

### Root cause (traced)
- 全域 APP_GUARD = JwtAuthGuard (app.module.ts:58)。每 endpoint 都過。
- JwtAuthGuard (modules/auth/guards/jwt-auth.guard.ts): JWT valid → isAdmin(internal_user_role∈admin/super_admin/system_admin) 全放 → 否則 validateDataAccess。
- list/approve/reject 不帶 customer_id/clinic_id → validateGeneralAccess(line149-157) 要求 accessibleCustomerIds 或 accessibleClinicIds 非空 → SD 兩者皆空 → line156 throw 403。← 就是 ticket 的錯誤。
- 第二層: controller 內 (integration-request.controller:293-321 advanced-search 用 customer_list; ~1084 validateCustomerAccess)。SD 在 advanced-search 區塊 fall through 不 throw; by-customer 路由的 validateCustomerAccess 仍會 throw (非核心 review 流程)。
- secret: 系統唯一 secret = JWT_SECRET (簽 JWT)。無另一套 API-key。

## Decisions Made
### [2026-06-15] Leo 決策: 有效 JWT 就給過
- 在 integration-management endpoint 跳過 guard 的 customer/clinic 檢查,仍要求有效 JWT。任何有效 token 者可用(比 SD-only 寬,Leo 接受)。範圍限這組 endpoint → 不違 AC3。
- 作法: 仿 @Public 做 @SkipDataAccessCheck() decorator (SetMetadata),guard 讀 reflector 命中就 return true (跳過 validateDataAccess)。
- 套用範圍(核心三組,review 時跟 Leo 確認是否擴及全部 8 個 integration-management controller): integration-request, status-management, vendor-inquiry。

## Retrospective

## Lessons Learned
- emr-v2 授權是**雙層**: (1) 全域 JwtAuthGuard (customer/clinic data-access + isAdmin) (2) controller 內 validateCustomerAccess/validateClinicAccess/validateIntegrationAccess。RBAC 類改動要兩層都處理。
- 區分兩種 gate: customer/clinic ownership (可用 skipDataAccess 放行) vs admin-only mutation (if(!isAdmin) throw,屬管理權限不該被 skip 連帶放寬)。
- 系統用 internal_user_role 區分內部/外部 user;validateIntegrationAccess 已對 internal_user_role 放行。未來「只給某內部角色」可走 internal_user_role 判斷而非全開。
