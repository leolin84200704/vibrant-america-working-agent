## Ticket Analysis

### [2026-06-10 21:59]
- **Title**: Failed to submit new integration request (Bug, P1-High, assignee Leo)
- **Symptom**: POST `/lisapi/v1/lis/emr-service/api/v1/integration-management/auto-integrate/requests` 回 400。
  Payload `ehrVendorId="4"` (非 Cerbo), `technicalRequirements.folder=""`。
  Response: `folder must be alphanumeric, dash, or underscore` + `must be longer than or equal to 1 characters`。
- **Reporter request**: 只有 Cerbo vendor 需要 folder，其他 vendor 應移除此限制。
- **Repo**: lis-backend-emr-v2, module `src/modules/integration-management/auto-integrate/`
- **相關歷史**: folder 欄位由 VP-16629 引入（Cerbo SFTP path 模板 `{folder}` 替換）。

### Root Cause
- DTO `TechnicalRequirementsDto.folder` (`dto/create-integration-request.dto.ts:136-140`):
  `@IsOptional() @IsString() @Length(1,100) @Matches(/^[a-zA-Z0-9_-]+$/)`
- class-validator `@IsOptional()` 只在值為 `null`/`undefined` 時跳過驗證；**空字串 `""` 不會被跳過** → `@Length(1,100)` 與 `@Matches` 觸發 → 400。
- Frontend 對非 Cerbo vendor 仍送 `folder: ""`，故全部撞牆。
- **Service 層其實已正確做 vendor 條件處理** (`services/integration-request.service.ts:120-142`):
  Cerbo → folder 必填 (缺則 throw 明確訊息); 非 Cerbo → 忽略 folder，用 template 既有 path。
  問題純粹在 DTO 層的無條件格式驗證擋掉空字串。

## Approaches Considered

### [2026-06-10 21:59]
- **方案 A（推薦，最小改動）**: 在 folder 欄位加 `@Transform` 把空字串/純空白正規化為 `undefined`，使 `@IsOptional()` 生效。
  - 非 Cerbo 空 folder → undefined → 通過 → service 用 template path。
  - Cerbo 空 folder → undefined → DTO 通過 → service throw 明確「folder is required for Cerbo」。
  - folder 有值時 → `@Length`/`@Matches` 仍生效，格式仍受保護（folder 會被代入 SFTP 路徑，格式驗證不可移除）。
  - `Transform` 已 import (line 15)，零新依賴。@Transform 在 ValidationPipe plainToInstance 階段即執行，驗證前生效。
- **方案 B（過度設計，不採）**: 自訂 `@ValidateIf`/custom validator 在 DTO 內查 DB 判斷 vendor 是否 Cerbo 再決定 folder 必填。
  - 缺點：與 service 既有邏輯重複、DTO 內做 async DB lookup、複雜度高。違反 Confirm Before Over-engineering。
- **方案 C（不採）**: 直接拿掉 `@Length`/`@Matches`。會喪失 folder 格式保護（SFTP path injection 風險）。

## Decisions Made

### [2026-06-10 22:00]
- Leo 選方案 A：DTO 放行（@Transform 空字串→undefined），service 把關 Cerbo 必填。
- 釐清關鍵：DTO 層只有 ehrVendorId，判斷 Cerbo 需查 DB，故 vendor 條件不放 DTO（留在已實作的 service）。
- Scope：只動 folder，msh06ReceivingFacility 同類潛在問題本 ticket 不碰。

## Lessons Learned

### [2026-06-10 22:15]
- class-validator `@IsOptional()` 只跳過 null/undefined，空字串仍驗證 → `@Transform` 正規化空字串為 undefined。（→ patterns.md）
- 條件驗證依據需查 DB 時別放 DTO custom async validator，留在 service 層；DTO 僅放行空值。（→ patterns.md）
- 測試失敗先 stash 比對 baseline 確認 pre-existing/regression，不口頭斷定。
