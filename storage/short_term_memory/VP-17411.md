SHARED charge block, and for stripe-first customers the charge now returns 2xx
requires_confirmation with EMPTY payment_id. Finalizer success check was only
`!errorMessage` -> success branch -> HL7 shipped unpaid with NO fail reason (visibility
regression vs the old 400) and consumed the unconfirmed intent's sample id.

Fix: success requires payment_transaction_id; otherwise fail reason
'payment failure: charge did not complete (no payment transaction id)' + warn, response
sample id NOT consumed, HL7 ships (Java parity) but visible, API path blocks via existing
guard. Stax (prod majority) byte-identical. +2 tests; 18 suites/165 green.
