# VP-17408 — Modernize 24 quarantined stale test suites

## Work record

### [2026-07-13 20:30] DONE — PR #258 (base staging)

**Method**: 4 parallel general-purpose subagents, one per module group with disjoint file
sets (vendor/status/config 6, auto-integrate+module 8, result 8, auth+hl7 2), all in the
same worktree/branch; jest.config quarantine removal + shared-util fix + integration
verification done by the main loop. Rule given to agents: current code = source of truth;
removed-feature tests deleted with markers; no source edits; no jest.config edits.

**Result**: 82/82 suites, 864 passed / 15 skipped, identical parallel and --runInBand.
(before VP-17407: 22-25 failing; after #256: 58 running/24 quarantined)

**Also fixed**: src/test-utils/mock-authenticated-user.ts (userId mirrored into required
user_id) — was broken for any importer.

**Findings requiring human/team follow-up** (documented in spec comments + PR #258):
1. LIKELY REAL BUG: ConfigurationManagementService.updateConfiguration writes
   `customMappings` (service line ~133) — column exists in NO schema, so every HL7
   configuration update dies at the Prisma layer. One-line fix + needs its own ticket.
2. OBR-16 provider name order: code emits NPI^Last^First; legacy reference HL7 says
   NPI^First^Last. Affects delivered HL7 results — needs ruling (check vendor complaints?).
3. schema.test.prisma uses Int for boolean columns -> IntegrationRequestService
   create/findAll untestable against SQLite; TestDataFactory passes booleans (broken).
4. TestHelper.createServiceTestSetup double-instantiates TestDatabase (reset() hits the
   wrong file).
5. IntegrationRequestService.update ignores paymentMethodId though the DTO accepts it.

**Behavior drift encoded in the new specs** (for future readers): 4-status
IntegrationStatus + PENDING->LIVE fast-track valid (VP-16629); NEW_INTEGRATION is an
integration_origin; EhrVendorApplicationService trimmed API; clinicId required numeric
string / contactPhone optional; ResultGenerationService missing-integration = hard fail
(VP-17342 guard) not default-config fallback; Cerbo compression threshold 6MB;
JwtStrategy accepts userId OR user_id; getTestCodeMapping rename.
