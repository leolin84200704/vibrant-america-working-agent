# VP-17408 — Modernize 24 quarantined stale test suites

## Work record

### [2026-07-14 09:05] STACKED-PR TRAP: #259 merged into the feature branch, NOT staging — rescue PR #261

- Merge order went parent-first: #258 (base staging) merged 00:48, then #259 (base
  bugfix/leo/VP-17408) merged 00:50 INTO THE FEATURE BRANCH. GitHub only retargets an
  open PR when its base BRANCH IS DELETED — it did not happen, so the three fixes never
  reached staging (verified: origin/staging lacks the OBR-16 change; branch tip 2edb0cc
  has it). Staging deploy 004cd36 = #258 only.
- **Rescue: PR #261** (staging <- bugfix/leo/VP-17408) carrying exactly the #259 diff.
- **LESSON (stacked PRs)**: merge child-first, or delete the parent branch on merge so
  GitHub retargets, or manually retarget the child to staging before merging. Verify
  `git log origin/staging` after every merge round — "MERGED" on the PR is not "on
  staging".
