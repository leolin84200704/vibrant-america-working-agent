**Title**: [BE] Map timezone from PST format to IANA format (e.g. America/*)
**Reporter**: Xiaoye Li (frontend VI/VA). **Assignee**: Leo. **Status**: Dev To Do. **Priority**: P2.

需求: Leo 的 service 回傳縮寫 timezone (PST)，FE (VI/VA) 與 Fan's service 用 IANA (America/Los_Angeles)。要 map 成 IANA。
AC: (1) service 回傳 IANA (2) 縮寫格式不再被回傳。

**Repo**: LIS-transformer-v2 (calendar/setting timezone area，延續 VP-16943/VP-16945/VP-17190)。

### Code audit (verified by reading files, not just explore agent)
Timezone setting 來源 = core SettingService (gRPC) 的 `timezone` setting，存的是縮寫 (PST/EST/...)。
已存在 mapping util: `src/calendar/models/shared/timezone.util.ts` → `resolveIanaTimezone(value)` (ABBREV_TO_IANA map, 預設 America/Los_Angeles)。

所有回傳 timezone 的出口：
- **GraphQL `getTimezoneSetting`** → user-settings.service.ts:75 `resolveProviderTimezone()` → IANA ✓ 已正確 (VP-17190)
- **REST `GET /getTimezoneSetting`** → setting.service.ts:4949 `findSettingValue(...) ?? 'PST'` → **回傳 raw 縮寫 ❌ 唯一洩漏點**
- schedule / public_booking / event / zoom / outlook / meeting-request 全走 `resolveProviderTimezone()` → IANA ✓
- reminder.service.ts:84 內部 read raw ?? 'PST' (email 格式化用，非 FE 回傳；VP-16945 範圍)
- Writer `setTimezoneSettingCustomer` (setting.service.ts:4972/4996) → 存 raw `timezone` 值未正規化

**結論**: FE 看到 PST 必定來自 REST `/getTimezoneSetting`。GraphQL 版早已轉 IANA。
