## Lessons Learned

1. **Calendar DB 操作第一步永遠 enumerate schema**：`SELECT schema_name FROM information_schema.schemata WHERE schema_name LIKE 'calendar%'`，不要假設目標 schema 是哪個。`calendar_dev`、`calendar_dev_new`、`calendar_prod` 同 DB 共存。
2. **query 結果空時診斷順序**：(1) `current_database()` / `current_schema()`，(2) `SELECT count(*) FROM <table>` sanity check，(3) 才懷疑 query 邏輯。
3. **DB backfill 必用 transactional + DO block verify pattern**：BEGIN → UPDATE → DO block 比對 expected mapping → 若不符 RAISE EXCEPTION 自動 ROLLBACK → 否則 COMMIT。比手動 SELECT 後 commit 更可靠。
4. **Zoom URL 寫入欄位**：`v2_event.external_url`（完整 URL 含 pwd）；`zoom_event_id` 是 meeting ID 不是 URL；`location` 是人類可讀 text，不是 URL 專用。
5. **Reminder template fallback chain（reminder.service.ts:237）**：`location || zoom_event_id || external_url || ''` — backfill external_url 時若 location 已有非空值，reminder 仍會顯示舊 location。這是 9181 的決策點。
