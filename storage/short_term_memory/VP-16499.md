## Lessons Learned

### [2026-05-26]
1. **Calendar DB еӨҡ schema**пјҡеҗҢдёҖ host `lis-postgresql.postgres.database.azure.com:5432`пјҢprod=`calendar_prod`гҖҒstaging=`calendar_dev_new`пјҢtable зөҗж§ӢзӣёеҗҢгҖӮд»»дҪ• schema и®Ҡжӣҙе…©йӮҠйғҪиҰҒеҒҡгҖӮ
2. **Idempotent migration `current_schema()` йҷ·йҳұ**пјҡ`IF NOT EXISTS (SELECT FROM information_schema.columns WHERE table_name=...)` дёҚеҠ  `table_schema = current_schema()` жңғи·Ё schema иӘӨеҲӨ вҶ’ з¬¬дәҢеҖӢ schema ж°ёйҒ  skipгҖӮ
3. **psql жІ’иЈқ**пјҡз”Ё `prisma db execute` жҲ– `new PrismaClient({ datasourceUrl })` + `$executeRawUnsafe` и·‘ raw SQLпјӣеҫҢиҖ…иҰҒиҮӘе·ұ split statementsгҖҒдҝқз•ҷ `$$` DO blockгҖӮ
4. **`npm run lint` её¶ `--fix`** вҖ” жңғ auto-reformat ж•ҙеҖӢ repoгҖӮи·‘д№ӢеүҚиҰҒиӯҰиҰәпјҢеҗҰеүҮжұЎжҹ“ branch 100+ unrelated filesгҖӮж”№з”Ё `npx eslint <files>` еҸӘжӘўжҹҘиҮӘе·ұзҡ„ fileгҖӮ
5. **legacy portal-calendar specialty**пјҡsource of truth жҳҜ Java service `/clinicians/first-available`пјҢBearer JWT authгҖӮpractice 150105 зҡ„ 9 еҖӢ clinician жҜҸдәә ~48-50 еҖӢ specialtyпјҢдё”зӣ®еүҚ**е…ЁйғЁ universally held**пјҲfilter ж©ҹеҲ¶ work дҪҶ prod data з„ЎеҚҖеҲҶж•ҲжһңпјүгҖӮ
6. **availability query жҳҜ UX provider дёҚжҳҜ conflict guard** вҖ” server-side `validateSlotAvailability` е·І hard-block double-bookingпјӣavailability query йЎҚеӨ–еӯҳеңЁжҳҜзӮәдәҶи®“ FE дё»еӢ•еҲ—еҸҜйҒёжҷӮж®өгҖӮ
