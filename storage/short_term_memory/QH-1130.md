### 建議方案（改前 / 改後為何有效 / 改什麼）
採方案 B（純驗證 + 補測試/文件，不改任何已 Done 的 endpoint 行為）。

改前現狀（已 grounding 程式碼）：LIS-transformer-v2/src/utility/ 已實作完整 — utility.controller.ts 共 36 個 REST endpoint、utility.service.ts 159KB、utility.dto.ts、utility.service.spec.ts 約 37 個 it/test。v1 LIS-transformer/src/utility/utility.controller.ts 有 74 endpoint（含大量 out-of-scope）。

關鍵修正（推翻 engineer 部分假設）：
1. 文件落後於程式碼，非相反。Confluence pageId 2197127170「Utility REST APIs (Trans v2)」(lastModified Apr 08) 只記錄到第 29 條，且缺 criticalReadoutAuthorizedContacts 系列、createPatient、updatePatientInfo、refreshToken。本機 README.md 較新（含 ~31 routes）。engineer 說「比對 Confluence 看 v2 有無遺漏」方向錯了 — 要做的是「用 controller 36 個當 source of truth 反向補 Confluence/README 缺的 ~6-7 條」。
2. 測試真正缺口比「偏薄」嚴重得多：spec 的 describe 只覆蓋 ~22/36 endpoint，且對 criticalReadoutAuthorizedContacts(GET/POST/update/delete)、createPatient、updatePatientInfo、refreshToken 這 6 個是 0 測試（grep 確認）。這些正是寫 DB（createPatient、criticalReadout 寫 contacts、muteSpecificEmail 寫 trigger_history）與發 token（refreshToken/getSettingToken）的高風險面。

改後為何有效：以 v1 controller 為 oracle 逐 endpoint diff request/response 契約，把未覆蓋 endpoint 補 spec（含非 happy path），文件補齊到 controller 實際 36 條，符合「v1→v2 parity 要逐 endpoint enumerate、cover 非 happy path」「test before push」鐵律，全在測試/文件層、不動 prod endpoint 行為，風險最低。

具體改什麼（全在 LIS-transformer-v2/src/utility/，不動 controller/service 邏輯）：
1. utility.service.spec.ts 補 0-coverage 的 6 個 endpoint（criticalReadout x4、createPatient、updatePatientInfo、refreshToken），優先 DB-write 與 token 路徑；既有 22 個補非 happy path（auth 缺失、accession_ids JSON parse 失敗、internal_call_code>=400、403 clinic 不符、404 sample lookup 失敗、shipSkinCare Billing Issue 400）。
2. README.md / Confluence 2197127170 補齊到 controller 實際 36 條（含上述缺的）。
3. 流程：branch bugfix/leo/QH-1130；切 branch 後先 npx prisma generate（雙 schema）；npm run start:dev 必過、npm run test 涵蓋新增分支必過後才考慮 PR base stage_test（不 direct push）。不動 src/trans/utility.resolver.ts（HubAPI，不同 scope）。

注意：start:dev / prisma generate 在目前 branch feature/leo/VP-16513 上做會污染，須先切 QH-1130 branch。

- createPatientBatchv2 每筆 sleep ~1s 並對 upstream 400 歸類為 duplicate；批次行為須驗 100% rows，勿 spot check
- Confluence(29) vs README(~31) vs controller(36) 三方不一致；以 controller 為 source of truth，勿照過時 Confluence 補成『刪掉 controller 多出的 endpoint』
- QH-919(1st batch, tests VP-13759) 仍 Dev To Do，1st batch 尚未 dev-complete 但 2nd batch 已 Done — 序列倒置，與 QH-1130 共用 src/utility/，補測試/文件範圍須與 QH-919 對齊避免重工或衝突
