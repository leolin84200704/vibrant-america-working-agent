## Proposed Action (GATED - prod INSERT to ehr_integrations)
INSERT one clinic-level RESULT_ONLY catch-all row: customer_id='-1', clinic_id=18299, ehr_vendor_id=6, mirroring the 142500 template; clinic_name="Spectra Healthcare Associates, PA". Keep the two provider FULL_INTEGRATION rows (they handle ordering + own results; same destination -> deduped, no duplicate delivery).

### Open decision — MSH-6

Adding a -1 row with msh06=18299 makes new providers send MSH-6=18299 while 11295/18373 still send 11295 (mixed). Options: (a) -1 row msh06=18299, leave provider rows; (b) align whole practice to 18299 (update provider rows too) — confirm Athena accepts 18299 first. Flagged for Leo/PM.
