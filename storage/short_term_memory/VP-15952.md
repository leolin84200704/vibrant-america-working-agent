## Retrospective

### [2026-04-17 19:05]
**1. 結果比較**
初始假設: 標準 EMR integration，用 insert-ehr-integration.ts script
最終結果: DocVilla 是新 vendor，script 無法用，需手動 SQL + SFTP 驗證
Scope 變化: 增加了 ehr_vendors 新增 + SFTP 連線驗證

**2. 失敗分析**
失敗總數: 2（輕微）
- Prisma tagged template SQL 語法問題（cuid 參數化失敗）→ 改用 executeRawUnsafe
- ehr_vendors INSERT 缺少 supported_hl7_versions → 加上欄位
兩個都是技術細節，快速修復。

**3. 模式識別**
- 64.124.9.100 是我們的共用 SFTP server，不同 vendor 用不同 port（Breathermae=2222, FTP=2224, DocVilla=2225）
- 新 vendor 需要先驗證 SFTP 連線，確認 port 和目錄結構
- insert-ehr-integration.ts 需要 ehr_vendors 中存在 vendor code，新 vendor 無法直接用
- updated_by / last_modified_by 欄位要填 Leo

**4. 教訓**
技術: Prisma $executeRaw tagged template 不支援 JS 變數直接插值到 VALUES，需用 $executeRawUnsafe + ?
流程: 新 vendor 的完整流程 = SFTP 驗證 → ehr_vendors 新增 → ehr_integrations 插入
業務: 我們的 SFTP server (64.124.9.100) 上不同 vendor 配不同 port，需先確認正確 port

**5. 信心度: 4/5**
理由: SFTP 驗證和 DB 操作都順利完成。唯一扣分是忽略了 updated_by/last_modified_by 欄位

## Lessons Learned

### [2026-04-17 19:05]
- 新 vendor 流程: SFTP 驗證 → ehr_vendors 新增 → ehr_integrations 插入
- 64.124.9.100 是共用 SFTP server，每個 vendor 有獨立 port
- insert-ehr-integration.ts 無法處理不在 ehr_vendors 中的新 vendor
- ehr_vendors.updated_by 和 ehr_integrations.last_modified_by 必須填 Leo
- Prisma $executeRaw 的 tagged template 不能直接插 JS 變數到 VALUES，要用 $executeRawUnsafe + ? 參數
- ehr_vendors 的 supported_hl7_versions 欄位是必填（JSON 格式）
