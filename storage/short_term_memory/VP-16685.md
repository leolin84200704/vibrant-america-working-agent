## Lessons Learned

### [2026-05-22] (詳見 [[INCIDENT-2604156666]] retrospective)
1. Cloud port 一定跟 on-prem port 一致；v1/v2 是 parallel services 不是 upgrade — 不要因為遷雲而 swap version
2. Pre-swap parity check: `grpcurl` 兩端跑同一 input、byte-compare response（含 timestamp/zero-value 等隱形欄位）
3. `tryCloudThenOnPrem` + kill switch 是 safe migration pattern：cloud primary、transient error 才退到 on-prem、business error 不退（不掩蓋真錯）
4. Scope 模糊時不要拍腦袋擴張，請 PM 在 Jira 上明確 deferred 項目（Redis/Kafka）
