### 建議方案（改前 / 改後為何有效 / 改什麼）
採最小可獨立交付切片：先補測試 + 寫文件，production code 不動（或僅限零行為變更的型別/lint 整理，每步跑 test）。

改前現狀（已驗證）：src/utility/ 已存在且龐大——utility.controller.ts 1108 行 / 36 個 REST endpoint decorator；utility.service.ts 4095 行 / 49 個 async 方法；utility.service.spec.ts 1255 行但只有 23 個 describe / 37 個 it，約 30+ 方法無測試。docs/ 只有 vp-16410-accession-claim.md，無 utility 設計文件。舊 v1 對應在 src/trans/utility.resolver.ts + src/trans/services/utility.api.service.ts。

改後為何有效：先用 characterization test 把現有行為「釘住」，是任何後續 refactor 的前置安全網（4095 行巨檔在 portal 線上流量上無 e2e 網下直接大改違反 Safety First）；文件化把 36 endpoint 的 input/output/上游依賴記下降低維護成本。此切片不依賴任何 sibling 票，可繞過 epic 的排程 block 立即交付。

具體改什麼（branch feature/leo/QH-919，base = stage_test）：
1. 切 branch 後先跑 npx prisma generate（兩份 schema：default 與 prisma2/schema2.prisma），確認 npm run start:dev / npm run test baseline 綠（repos.md 鐵律，否則 test 會炸）。
2. 為未覆蓋的 service 方法補 unit test，優先線上高頻：createPatient、updatePatientInfo、updateCustomerProfileV4、refreshToken、updateCollectionStatus、createPatientBatchv2 邊界、critical-readout CRUD（create/update/delete/listCriticalReadoutAuthorizedContact(s)）、checkPatientCustomer、hasBillingIssue。
3. 新增 docs/qh-919-utility-module.md：列 36 endpoint、對應 service 方法、上游 gRPC/REST 依賴；若 PM 要 Confluence 則同步 pageId 1602355225「back log for transv2」。
4. 不動 production code（或僅零行為變更整理，每步 test）。
5. 全英文 code/comment（no-CJK 鐵律），commit [QH-919] ...，PR → stage_test（禁直推 staging/main）。

  - 取捨：已被 PM 2025-11-24 留言明確推翻（not rewrite using GraphQL），照做等於做錯方向、白工。

### 風險與邊界案例
- 線上高頻 endpoint（createPatient/refreshToken/updateCustomerProfileV4/updateCollectionStatus）若被 refactor 動到行為，影響 portal 線上流量——故本批 code 不動，先以 characterization test 鎖行為。
- characterization test 必須對真實依賴行為建模（live-not-mock 精神）：若只 mock 上游 gRPC/REST 回傳假資料，會把錯誤行為當正確釘住；應比對既有實際 response shape。
- 「1st Batch」範圍未定義——36 個 endpoint 哪幾個算第一批沒講清，未確認會做錯範圍。
- 切 branch 後忘了 npx prisma generate（兩份 schema）會導致 start:dev/test 紅，被誤判為 pre-existing 失敗（start:dev iron rule）。
- QH-919 的 Automation 留言誤報 VP-13759「has been resolved」，實際 VP-13759 = Dev To Do、epic = Dev Blocked；別被此 stale 自動通知誤導以為已完成。
