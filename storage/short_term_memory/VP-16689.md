**假說 B：Claim table 有 2604096219、但 resolver 因 normalize/比對失敗找不到**

- Schedule 端 createEvent 走 prisma client 寫進 array，可能格式不同
- **驗證**：DB 查 `v2_event_accession_claim WHERE accession_id LIKE '%2604096219%'` 看實際存的字面

**假說 C：Event 是用 `THIS_EVENT` scope 取消（exception row）→ 未 release claim**
- VP-16410 STM 已記載 THIS_EVENT 不動 master，hook 沒覆蓋 — 我當初標記為 out-of-scope 等 PM 確認
