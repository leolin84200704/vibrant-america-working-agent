## Lessons Learned

### [2026-05-28 18:10]

**技術面**
1. **PR merge conflict 解析 hybrid 策略**：純格式差（prettier `.then().catch()` chained vs flat）採 HEAD（保留 feature branch 的 lint cleanup commit）；實質 code change（cast、eslint disable 註解、blank line）採 incoming branch（保留 upstream 的 type-safety 補強）。這個策略適用於 base branch 大規模 PR merge 進 feature branch 的場景。
2. **`buildSyncAttendeeIds` 回傳型別 narrow 比 caller-side disable 乾淨**：當 `Promise<any[]>` 被多處（13 caller）當 `Customer[]` 用，最 clean fix 是改 source function 回傳 `Promise<Customer[]>` + cast `as unknown as Customer` 集中在 source 一行；不是在每個 caller 加 eslint-disable。前提：sync service 內部只用 partial fields（`.id`），結構上是 type-laundering，functionally 等價。
3. **`logger.error(msg, error)` → `logger.logError(msg, error)` 是 file convention align 不是 behavior change**：本 file 既有 54:13 比例，前者走 NestJS 標準 (`trace?: string` 強制 stringify)，後者走 LisLoggingService.sanitizeError（`error?: unknown` properly 拆 message/stack）。catch block 內統一用 logError 是既有 dominant pattern。
4. **`as unknown as T` 是 ESLint 認可的「我知道 type lying」escape**：直接 `as T` 會被 `no-unsafe-type-assertion` 抓（如果 enabled），`as unknown as T` 不會。用在 sync service signature 故意接 `Customer[]` 但實際只讀 `.id` 的場合。

**流程面**
1. **Step 1 Retrieve 第一動作要 grep failures.md**：start:dev / build 失敗時，pattern keyword（`MODULE_NOT_FOUND`、`Cannot find module`、`prisma2.*client2`、`start:dev`）一定要先 grep failures.md。VP-16410 翻過的車，failures.md 既有 entry，但這 session 第一輪走 4 個 option spike → 浪費一輪 tool call。
2. **dist 結構先看頂層 `ls dist/`**：tsc/nest build 路徑解析有關 bug，先看 `dist/` 頂層結構（有沒有 `dist/src/...` 嵌套），比 grep import path 更快定位。
3. **merge in-progress 時禁用 git stash**：`git stash push` 會把 MERGE_HEAD 跟 conflict markers 一起收走，pop 回來變 needs-merge 狀態無法直接 apply。要保存 in-flight diff 改用 `git diff > /tmp/wip.patch`，或先 commit merge 再 amend。

**工具面**
1. **mcp__ide__getDiagnostics 不可靠**：WebStorm 抓的 lint complaint = `npx eslint <file>`。agent 端跳過 IDE diagnostic round-trip，直接 CLI 拿結果。
2. **`eslint --fix` 會自動移除 unused eslint-disable directive**：解 conflict 採 incoming 的 disable 註解後，跑 `--fix` 可能自己把它拿掉（因為 narrow type 後 rule 不再 trigger） — 不用手動清。
