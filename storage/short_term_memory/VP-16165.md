## Lessons Learned

### [2026-05-05 22:40] 技術
- **MSH.4 解析索引**：lis-backend-emr-v2 的 `Hl7DecoderService.parseSegments` 對每個 segment 用 `split('|')` 後 `slice(1)`。因此 MSH segment 的 `fields[0]` = MSH.2（encoding chars `^~\&`），`fields[2]` = **MSH.4** (Sending Facility)。寫類似「抽 MSH.X」的 helper 時要用這個 mapping。
- **Defensive HL7 parsing**：抽 MSH 任一欄位前先驗 `fields[0]` 是否含 `^`（encoding chars marker）；異常就 warn + return null，避免錯把鄰近欄位當目標欄位。
- **lis-backend-emr-v2 的 `ehr_integrations.clinic_id` 是 `Int?`**（非 String），對 MSH.4 比對前要 `Number()` 轉並驗 `Number.isInteger`。

### [2026-05-05 22:40] Repo Pattern
- **Clinic-level catch-all 慣例**：lis-backend-emr-v2 用 `customer_id = '-1'` 在 `ehr_integrations` 標記 clinic-level integration（`'-1'` 是 string 因為 customer_id 是 `VarChar(255)`）。將來 `practice_integrations` 表獨立後此慣例會搬遷。

### [2026-05-05 22:40] 流程
- **Step 2 Epic 調查必查 sibling**：JQL `parent = <epic>` 列出所有兄弟 ticket，看誰 own 哪個 phase / dependency 順序，避免錯認本 ticket 範圍。
- **Step 4 呈報方案時要主動 propose「最小可獨立交付片」**：不要只列「完整 vs 大合併」兩極，補一個「拿掉所有 dependencies 後的 thinnest slice」常常是 Leo 偏好。
- **Confluence PRD 過大時**用 `mcp__claude_ai_Atlassian__fetch` (markdown ARI form) 取代 `getConfluencePage`，後者 ADF JSON 易爆 token 限制。

### [2026-05-05 22:40] 工具
- `mcp__claude_ai_Atlassian__fetch` 的 ID format: `ari:cloud:confluence:<cloudId>:page/<pageId>` — 直接傳純 page ID 會 fail。
