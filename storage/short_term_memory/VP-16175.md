## Lessons Learned

### [2026-04-17 18:10]
- RESULT_ONLY integration 不需要 order_clients（update-order-clients = false）
- Prisma model: camelCase（ehrIntegration）; DB table: snake_case（ehr_integrations）
- Athena integration 是最簡潔的 vendor 之一：不需要 sftp_folder_mapping，用 vendor 預設即可

### [2026-05-20 17:30] 追溯教訓（最重要）
- **Batch DB INSERT 驗證必須 100% 檢查每一 row 的每個關鍵欄位（status / type / vendor_id / clinic_id）**，不是 spot check 抽幾筆就好
- **特別小心：第一筆 vs 後續筆參數可能不同** — script CLI 第一次跑可能漏帶 flag、Leo 在中途糾正後續才對。如果只看「全部都 inserted、status 看到 LIVE」就放心，會漏掉前面預設值的 row
- **Prisma schema `@default(PENDING)`** — `insert-ehr-integration.ts` 如果沒顯式傳 status，會落到 PENDING；ehr_integrations.status enum 是 PENDING / APPROVED / LIVE / REJECTED
- **驗證 SQL 模板**：每張 ticket 跑完都用
  ```sql
  SELECT customer_id, status, integration_type, ehr_vendor_id, clinic_id, created_at
  FROM ehr_integrations
  WHERE customer_id IN (...ticket 列出全部...) AND clinic_id = {practice_id}
  ORDER BY customer_id;
  ```
  逐 row 對照預期值，**不放過任何一筆**
