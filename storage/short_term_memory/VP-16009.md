**問題描述**:
在 Vibrant Intelligence 的 Personal Settings > Third-party Integrations 頁面，出現 "Failed Network Error"（CORS 錯誤）。

**失敗 URL**: `GET https://www.vibrant-america.com/lisapi/v1/lis/emr-service/api/v1/integration-management/auto-integrate/requests?customerId=999997`

**涉及元件**:
- Frontend: va-portal (Vibrant Intelligence)
- Backend: lis-backend-emr-v2 (`auto-integrate/requests` endpoint)
- Infrastructure: lisapi 反向代理 / API Gateway

**Comments 摘要**:
1. Jira Agent 建議可能涉及 LIS-backend-v2-coreSamples, va-portal, LIS-Report
2. Leo 問 Kristine 提供 website URL（2026-04-08）
3. 自動化通知 Dev Blocked 狀態

**Root Cause 分析**:
CORS 配置有多層，且存在 mismatch：

**Layer 1: K8s NGINX Ingress** (`k8s/base/ingress.yaml` L38):
只允許 2 個 origin:
- `https://portal.vibrantamerica.com`
- `https://admin.vibrantamerica.com`

**Layer 2: NestJS App** (`.env` ALLOWED_ORIGINS):
允許 6 個 origin:
- `https://localhost:3000`, `https://localhost:3001`
- `https://portal.vibrant-wellness.com`
- `https://portal.vibrantamerica.com`
- `https://admin.vibrantamerica.com`
- `https://www.vibrant-america.com`

**Layer 3: 外部 Gateway** (`www.vibrant-america.com/lisapi/...`):
未知配置

**關鍵問題**: Vibrant Intelligence (VI) 前端的 origin domain 不在任何 CORS allowlist 中。
