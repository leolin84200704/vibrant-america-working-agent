## Lessons Learned
### [2026-04-27]
**hl7_file_input triage 流程 (regular ops):**
1. 查詢: `parse_finished=0 AND retry_num=0 AND received_time >= NOW() - INTERVAL 72 HOUR`
2. 分類失敗原因:
   - `emr_code_not_found` 有值 → test code / bundle mapping 問題 → 報 PM
   - `order_input` 有值但 `sample_id=NULL` → 下單流程失敗 → 需手動 recovery
   - `order_input=NULL` 且 `emr_code_not_found=NULL` → HL7 parsing exception → 查 EMR-Backend logs
3. Recovery 手動流程:
   a. getAllPaymentMethods API 取得 payment_token + customer_token
   b. transactionPay API 扣款（必須帶完整 fields: currency, charge_type, token_platform, payment_source, new_sample）
   c. Order API 下單（用新的 sampleId, payment_id, julienBarcode）
   d. UPDATE hl7_file_input SET parse_finished=1, sample_id=X, payment_id=Y, retry_num=0

**VACP code 分析流程:**
- VACP{panelId} → custom bundle，查 bundle mapping API
- 先查 customerId+panelId，再查 clinicId+panelId
- 找不到 → 報 Order team / PM

**關鍵 API endpoints (prod):**
- Payment methods: GET `vibrant-america.com/lisapi/v1/charging/paymentMethod/allSharedPaymentMethods?customer_id=X&clinic_id=Y`
- TransactionPay: POST `vibrant-america.com/lisapi/v1/charging/transaction/pay`
- Order: POST `vibrant-america.com/lisapi/v1/portal/order/orderTest/order`
- Bundle mapping: GET `api.vibrant-wellness.com/v1/pricing/item/promotion/getLegacyBundleMapping?currency=usd`

**JWT token generation:**
- 用 prod secret 簽 HS256
- payment 用 role="clinic", getTokenCustomerPM=true
- order 用 role="customer"
- 固定值: userId=54674, internal_user_name="bolin.l", internal_user_role="admin"

**hl7_file_input 處理流程 (EMR-Backend ParseOrder):**
- 插入時 retry_num=5 (INITIAL_RETRY_NUMBER)
- Cron: SELECT WHERE retry_num > 0 AND parse_finished = false
- 成功: parse_finished=true (sample_id >= 0)
- 失敗: retry_num--, parse_finished 維持 false
- retry_num=0 時永久卡住

**sftpDir → clinic 反查:**
- hl7_file_input.sftpDir → order_clients.remote_folder_path → ehr_integrations.customer_id → clinic_id
