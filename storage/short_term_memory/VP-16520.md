### 需求（cancel-and-rebook orchestration）
- submission 時偵測「選的 clinician 是否 ≠ 原 assigned clinician」(F-3)
- **不同** → cancel 原 calendar event、原 appointment record 標 `cancelled`（**非** rescheduled）、在**新 clinician 行事曆建新 event**
- **相同** → update 既有 event in place（標準 reschedule, F-9）
- 新 event 必須 link 回原 patient/order record（F-8，consult history 不斷）
- atomic where possible、0 orphan、reporting 不把 switch-cancelled 當 open（AC6）
- 明寫「coordinate ordering with email triggers」= [[VP-16521]]

### 與 VP-16521 整合
- VP-16520 = event 編排（cancel 舊 + 建新 / update in place）；VP-16521 = email triggers 掛在編排步驟上
- email 掛點：switch → T8 掛 cancel-原-event（給舊 clinician）、T2 掛 create-新-event（給新 clinician）、T6 給 provider；same → T7+T6 掛 in-place update
- **重塑 VP-16521 已寫的 helper**：原假設 in-place 換 participant（同一 event 讀新舊 clinician）→ 改成掛在 cancel/create 兩步。helper 的 templateModel/變數邏輯可留用，hook 點要改。

## Lessons Learned

### [2026-05-27]
1. **Clinical Consult「換 clinician reschedule」= cancel-and-rebook,不是 in-place 換 participant**：原 event `is_canceled=true`（reporting 靠此 boolean filter,無 status enum）+ 新 clinician 行事曆建新 event,F-8 抄 creator_calendar_id/practice_event_type/accession_ids;同 clinician 才 in-place。VP-16520+16521 合一個 mutation `rescheduleClinicalConsult`。
2. **v2_event 無 status enum,只有 `is_canceled` boolean** 驅動 open/pending reporting。
3. **accession 1:1 claim（VP-16410）搬移**：switch 時同 transaction 內 **release 舊 → claim 新**（順序很重要,否則撞 unique）。`accessionClaim.releaseForEvent/claimForEvent` 都吃 `tx`。
4. **calendar reschedule email reuse**：switch 用既有 create(→新)+cancel(→舊)template,same 用 update template;provider 一律一封 reschedule(T6)。不能直接 call 既有 bulk send method（會多送 provider 信）→ 自寫 targeted 編排 helper。
5. **side-effects 比照 create/cancel/update**：Kafka `sendAppointmentEvent` + 外部行事曆 `syncEventToExternalServicesOnCreate/OnUpdate/deleteEventFromExternalServices` + Zoom delete/update,全 fire-and-forget。
6. **`npm run build` 的 18 `specialties` 錯誤是 stale generated prisma client**：切 branch 後 client 未重生成會誤報;build prebuild `prisma generate` 後即消。debug build 錯誤前先確認 client 是否對應當前 branch schema。
7. **LIS-transformer-v2 base branch = `stage_test`**（非 staging/main）;calendar DB = PostgreSQL。
