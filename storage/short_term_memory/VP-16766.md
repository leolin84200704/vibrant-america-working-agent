## Lessons Learned
### [2026-05-27]
1. **Optimantra FULL_INTEGRATION 標準範式**（VP-16193 + VP-16766 + 5 筆 LIVE 前例）：
   - `ehr_integrations`：integration_type=FULL_INTEGRATION, status=LIVE, msh06=Practice ID, sftp_result_path=`/Prod/Input/`, sftp_archive_path=`""`, **sftp_ordering_path=null**, report_option=PERSONALIZED, **kit_delivery_option=NON_BLOOD_ONLY**, legacy_emr_service=OPTIMANTRA, ordering/result/sftp_enabled=1, sftp_host=45.24.217.155:22
   - `order_clients`：kits_options=**0**, emr_name=`OPTIMANTRA`(大寫), remote_folder_path=`/Prod/Orders/`, old_clinic_id=null, customer_name 不含 suffix
   - `sftp_folder_mapping`：**不新增** — 全 Optimantra 共用 id=67（server=/Prod/Orders/, local=/OPTIMANTRA/Prod/Order/, use_v2_pipeline=1）
2. **Optimantra vs MDHQ folder 模式差異**：Optimantra = vendor 共用固定 folder（/Prod/Input/ + /Prod/Orders/），per-practice 不建 mapping；MDHQ = client-specific `/{folder}/...` 每 practice 一筆 mapping。判斷要不要動 sftp_folder_mapping 先看 vendor 是哪種模式。
3. **insert-ehr-integration.ts 對 FULL_INTEGRATION 有 kit 錯配 bug**：line 292 寫死 kit_delivery_option=NO_DELIVERY，但 line 420 kits_options=0（兩者不對齊）。FULL 用 script 須事後 UPDATE 修；single $transaction 可一次寫對，較適合 FULL_INTEGRATION。
4. **from-scratch new 不產生 status_history**：VP-16379（script insert）確認無 history 列；只有 stub finalize（PENDING→LIVE）才補 status_history。
