### 風險與邊界案例
- cardio_genetics_questionnaire 是 dead mapping：若 VP-13464 需求其實含 cardio genetics，純驗證會 silently 放過缺陷；cardio_genetics 在 setting 模組是真實 report type，提高它是真 gap 而非單純死碼的可能性
- 外部 status API（interactive-report / inventory）失敗時 service catch→null，questionnaire 仍輸出但 status/end_time 為 undefined：驗證需確認此 graceful 降級是預期行為而非吞錯
- validateInput 要求 sampleId 為正整數字串、accessionId 非空；邊界輸入（負數、非數字、空白）應回 BadRequest，需覆蓋
- Redis cache 600s：驗證同一 (sampleId, accessionId) 連續查詢命中 cache，且 cache miss/寫入失敗只 warn 不中斷
- PNS_BASE_URL / VA_QUESTIONNAIRE_URL / shippin_address / questionnaire_status / getQuestionnaireRequiredMap 等 env 在 stage 須正確設定，否則 URL 與 status 端點錯誤；驗證 stage config 與 prod 差異
- generateBackendToken 依 SERVER_ENVIRONMENT 切 secretOrKeyDev/Prod，跨環境驗證需確認用對 secret
