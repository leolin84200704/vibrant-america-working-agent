### 建議方案（改前 / 改後為何有效 / 改什麼）
目的: QA sign-off the existing result-sending API; do NOT write new production code (VIB-1641 already Done, surface is complete — verified in repo).

改前現狀: src/modules/result/ already exposes both REST and gRPC. REST controller.ts (13 routes, all @UseGuards(JwtAuthGuard) + user.isAdmin): POST generate/:sampleId (L129, body {send_result?, add_report?}), POST generate-content/:sampleId (L159, "forces no report, does not transmit", returns hl7_content inline), POST patient-hl7-content (L185, read-only, no file/DB/transmit, by patient + start/end time), POST generate/batch (L211, sample_ids[]), POST :id/transmit (L98), GET status/:sampleId (L237), GET sample/by-order/:orderid (L259), GET order/:orderId (L112), plus CRUD @Post/@Get/@Put/@Delete. add_report: "1"=classic, "2"=personalized, undefined=DB config (confirmed L144-145). gRPC (result-grpc.controller.ts, proto src/proto/result-generation.proto): GenerateResultHl7 / GenerateBatchResultsHl7 / GetResultStatus / GetSampleByOrderId / GetHl7Content; gated by isPusher pod-role (L87/L124 — non-pusher pod returns success:false and does nothing); gRPC responses deliberately omit hl7_content (>4MB hangs gRPC, L191/L201 — fetch via GetHl7Content RPC); error_details truncated at 10*1024=10KB (L232).

改後為何有效: Verifying the live API against a structured checklist gives the QA sign-off the ticket asks for without touching code; using read-only paths (generate-content / patient-hl7-content) and send_result:false avoids polluting prod or the vendor SFTP. End-to-end transmit, if PM authorizes, is diffed old-vs-new HL7 to catch the cross-language serialization class of bug (INCIDENT-2604156666 Go zero-value timestamp 00010101000000 polluted 990 results).

改什麼: No production code. Produce (1) a QA verification checklist and (2) endpoint documentation in team-standard structured markdown (per feedback_api_doc_format — NOT raw OpenAPI). Reuse existing docs at repo root /Users/hung.l/src/lis-backend-emr-v2/OPTIMALDX_RESULT_SENDING_SUMMARY.md and docs/RPC-IMPLEMENTATION-GUIDE.md, docs/GRPC-URLS.md, docs/add-report-parameter-implementation.md. Checklist must cover: each REST endpoint happy path; add_report three-state (1/2/undefined); generate-content & patient-hl7-content read-only guarantees (assert no file write, no DB write, no transmit); batch partial-failure behavior; gRPC isPusher gate (must hit pusher pod); GetHl7Content for large-payload path; error_details >10KB truncation; JwtAuthGuard + isAdmin enforcement (non-admin -> 403). Deliver verification as a draft Jira QA sign-off comment for Leo to post (never post directly, per feedback_jira_comment). If a real bug is found, file/handle separately — do not auto-expand this ticket's scope.

### 候選方案與正反取捨
- **Treat as a genuine new-build ticket and implement a fresh result-sending API**
  - 取捨：Wrong — duplicates fully-existing src/modules/result/ surface; VIB-1641 is Done; would conflict with shipped code. Rejected after confirming the API exists.
- **Opportunistically harden the API (input validation, observability, missing endpoints) while verifying**
  - 取捨：Over-engineering with zero AC backing; violates Confirm-Before-Over-engineering. Only act on code if QA surfaces a concrete defect, and then via a scoped fix/ticket.
- **End-to-end verify by actually transmitting to vendor SFTP (send_result:true) in prod**
  - 取捨：High blast radius — pollutes prod/vendor. Only with explicit PM approval, against test sample IDs, with old-vs-new HL7 byte diff and peer-observed SFTP confirmation.

### 風險與邊界案例
- send_result:true on generate/* really pushes HL7 to the vendor SFTP — verify with send_result:false / generate-content / patient-hl7-content (read-only) unless PM authorizes a bounded e2e test
- gRPC GenerateResultHl7 silently no-ops on non-pusher pods (isPusher gate L87/124, returns success:false) — QA must target the pusher pod or results look 'broken' falsely
- gRPC responses never carry hl7_content (>4MB hang); validating content requires a separate GetHl7Content RPC — easy to mis-conclude content is empty
- error_details truncated at 10KB (L232) — long stack traces appear cut off; not a bug
- Cross-language gRPC serialization坑: Go zero-value timestamp 00010101000000 previously corrupted 990 results (INCIDENT-2604156666) — any e2e check must byte-diff old vs new HL7, esp. date/time fields (collection date, MSH.7)
- add_report undefined falls back to DB config — verify the DB-config branch separately from explicit 1/2
- batch generate partial failure: confirm per-sample success/error reporting and that one bad sample doesn't abort the whole batch
- verify live not mock (feedback_verified_means_live_not_mock) — do not call mock unit tests 'online verified'; reproduce with real DB / dist service
